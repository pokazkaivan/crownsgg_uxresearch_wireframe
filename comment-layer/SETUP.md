# CommentLayer — підключення та деплой

Шар коментарів уже вшитий у чотири сторінки: лендінг, прототип і обидві специфікації.
Зараз він працює в **локальному режимі** — коментарі зберігаються тільки у браузері
того, хто їх залишив. Щоб команда бачила коментарі одне одного, треба під'єднати
Supabase і задеплоїти на Vercel.

Порядок саме такий: **спершу Supabase → потім config.js → потім Vercel**.
Якщо задеплоїти раніше, ніж вписати ключі, сайт працюватиме, але коментарі не будуть спільними.

---

## Крок 1 · Supabase (5 хв)

1. Зайди на [supabase.com](https://supabase.com) → **New project**.
   Безкоштовного тарифу вистачає. Регіон бери найближчий до команди (Frankfurt).
2. Дочекайся, поки проєкт підніметься (~2 хв).
3. Ліве меню → **SQL Editor** → **New query**.
4. Відкрий файл `comment-layer/schema.sql`, скопіюй увесь вміст, встав у редактор і натисни **Run**.
   Має з'явитися `Success. No rows returned` — це нормально.
5. Перевір: ліве меню → **Table Editor** → має бути таблиця `comments`.

Що робить цей скрипт: створює таблицю `comments`, індекс за проєктом, вмикає
row-level security з політикою «будь-хто з ключем може читати й писати» і вмикає
realtime, щоб коментарі з'являлися в інших вкладках одразу.

---

## Крок 2 · Ключі у config.js (1 хв)

1. У Supabase: **Settings → API**.
2. Скопіюй два значення:
   - **Project URL** — виглядає як `https://abcdefgh.supabase.co`
   - **anon / publishable key** — довгий рядок, починається з `eyJ…` або `sb_publishable_…`
3. Відкрий `comment-layer/config.js` і встав їх замість заглушок:

```js
window.COMMENT_LAYER_CONFIG = {
  supabaseUrl: "https://abcdefgh.supabase.co",
  anonKey:     "eyJhbGciOi...",
  projectId:   "crowns-gg-wireframe",
  readySelector: "#view > *, .wrap, main, article",
};
```

`anon key` — публічний за задумом, його безпечно тримати в коді фронтенду.
Ніколи не встав сюди `service_role` ключ.

`projectId` міняй тільки якщо треба почати рев'ю з чистого аркуша — коментарі
розділяються саме за цим полем.

---

## Крок 3 · Vercel (5 хв)

Найпростіший шлях — через GitHub, бо тоді кожен новий пуш автоматично оновлює прев'ю.

**3.1. Заливаємо в GitHub**

```bash
cd "Crowns GG/crownsgg_uxresearch_wireframe"
git init
git add .
git commit -m "Crowns wireframe + CommentLayer"
git branch -M main
git remote add origin https://github.com/ТВІЙ-НІК/crowns-wireframe.git
git push -u origin main
```

Репозиторій зроби **приватним** — усередині ключ Supabase.

**3.2. Деплой**

1. [vercel.com](https://vercel.com) → **Add New → Project** → **Import** свій репозиторій.
2. У налаштуваннях імпорту:
   - **Framework Preset** → `Other`
   - **Root Directory** → залиш кореневим, якщо ти залив саме теку
     `crownsgg_uxresearch_wireframe`. Якщо залив усю теку «Crowns GG», то
     тут вкажи `crownsgg_uxresearch_wireframe`.
   - **Build Command** → лишити порожнім
   - **Output Directory** → лишити порожнім
3. **Deploy**. За хвилину отримаєш адресу виду `crowns-wireframe.vercel.app`.

**Альтернатива без GitHub** — через CLI:

```bash
npm i -g vercel
cd "Crowns GG/crownsgg_uxresearch_wireframe"
vercel          # прев'ю
vercel --prod   # бойова адреса
```

## Крок 4 · Перевірка (2 хв)

1. Відкрий задеплоєну адресу.
2. Праворуч має з'явитися вкладка **Comment** (або натисни клавішу `C`).
3. Клікни на будь-який елемент → напиши текст → збережи.
4. Онови сторінку — коментар на місці.
5. Відкрий ту саму адресу в іншому браузері чи з телефона — коментар видно там теж.
6. У Supabase → **Table Editor → comments** має бути рядок з твоїм `project_id`.

Якщо пункт 5 не спрацював — дивись «Якщо щось не так» нижче.

## Що вміє шар коментарів

- Клік по елементу лишає пін із номером; є режими **Text** (фраза), **Multi**
  (кілька елементів в один коментар) і **Area** (виділити область).
- Кожен коментар зберігає скриншот елемента, його HTML і маршрут сторінки —
  тобто видно не лише «що написали», а й «до чого саме».
- Панель рев'ю: пошук, сортування, вкладки Open/Closed, resolve / reopen / delete.
- Гарячі клавіші: `C` — коментар, `P` — панель, `H` — сховати піни, `Esc` — вийти.
- Працює на телефоні: кнопка внизу праворуч, панель — на весь екран.
- Інтерфейс ізольований у Shadow DOM, тож стилі прототипу він не чіпає.

Коментарі спільні для всіх чотирьох сторінок — прототипу, лендінга й обох
специфікацій. Розрізняються за URL, який зберігається в кожному записі.

## Якщо щось не так

**Вкладки Comment немає взагалі**
Відкрий консоль браузера. Повідомлення `[CommentLayer] not loaded` означає, що не
знайшлися файли — перевір, що тека `comment-layer/` потрапила в деплой.

**У консолі «Local mode»**
Ключі в `config.js` лишилися заглушками або файл не оновився в деплої.
Перевір, що зміни закомічені й Vercel зробив новий білд.

**Коментарі не видно з іншого браузера**
Не виконався `schema.sql` або виконався не в тому проєкті. Перевір, що таблиця
`comments` існує, і що в **Settings → API** ти брав ключі саме з цього проєкту.

**У консолі помилки CORS або 401**
Скоріше за все, вставлений не той ключ. Потрібен саме **anon / publishable**,
не `service_role`.

**Локально з файлу нічого не зберігається**
Так і має бути: при відкритті через `file://` Supabase недоступний, тому вмикається
локальний режим. Для повноцінної перевірки використовуй задеплоєну адресу або
запусти локальний сервер: `python3 -m http.server 8080` у теці проєкту.

## Коли рев'ю завершиться

Коментарі можна вивантажити у структурований бриф для розробки:

```bash
SUPABASE_URL=https://abcdefgh.supabase.co \
SUPABASE_KEY=eyJhbGciOi... \
node scripts/export-comments.mjs crowns-gg-wireframe open
```

Скрипт лежить у репозиторії CommentLayer і збирає текст коментарів разом із
посиланням на елемент, HTML і скриншотами.

**Важливо:** політика бази відкрита — будь-хто, хто має адресу сайту, бачить і може
редагувати всі коментарі. Для внутрішнього рев'ю це нормально, але не давай
посилання назовні. Перед публічним запуском шар коментарів треба або прибрати
(видалити два `<script>` теги зі сторінок), або закрити авторизацією.
