Crowns — Site Map & User Flow Specification

The complete blueprint for the FigJam canvas and the interactive wireframe that follows. Every screen below is documented with its purpose, components & functions, access level, interactive states, modals, and all inbound/outbound connections — so each card maps 1:1 to a node you place on the canvas.

Real-world (IRL) physical items ~24 MVP screens Stage 2 layer flagged Auth: Email + Discord OAuth Cashier: Crypto + gift cards Withdrawal = marketplace ticket

00 How to read this document

Each screen frame carries a set of badges and four content blocks. On the canvas, treat each frame as one node/sticky group; treat each connection line as an arrow between frames.

ROUTE

A real URL/page the user can land on or deep-link to.

MODAL

An overlay on top of the current page — no URL change. Returns user to where they were.

PUBLIC

Viewable without an account. Drives SEO & first impression.

GATED

Requires login. Any action here triggers the Auth modal if logged out.

VIEW PUBLIC · ACT GATED

Anyone can browse; the value action (open / play / submit) requires login.

STAGE 2

Post-launch. Map it on the canvas as a dimmed/parked layer, not in the MVP build path.

Frame blocks:  Functions = components & actions on the screen (• core, ○ optional/nice-to-have).  States = every visual condition the wireframe must cover (empty, loading, win, lose, etc.).  Modals = overlays this screen can raise.  Connections = where the user comes from (In) and goes to (Out).

01 Product & design direction

Anchored in the client's answers and the competitor research. Crowns is an IRL physical-item case platform (not CS2 skins, not RuneScape). The aesthetic target is the "quiet" end of the spectrum — packy.gg / packdraw calm, ★★☆☆☆ — sharp and smooth, never scammy.

Decision areaDirection
Platform modelReal-world physical items. Won items go to inventory/cart → fulfilled by a human Crowns team via a marketplace ticket (no user-entered shipping at withdrawal).
Day-1 mechanicsCase openings (single + multi up to x4), demo spins, case battles, upgrader (Borrow mode up to 95%), rewards (monthly race / weekly challenges / raffle / free cases by level).
BrandStart fresh — drop the loud V1 purple. Pale or dark, restrained scheme. One accent; currency/results are the only other colors.
AuthEmail + Discord OAuth. Age gate 18+.
CashierCrypto + gift cards day 1. Apple Pay / fiat = Phase 2 (third party).
ProfileVIP tiers + affiliate dashboard important. Stats on request. Public Win Showcase reachable by clicking a username in chat/site.
SoundA rain.gg × packdraw blend; original assets are Stage 2.

Visual language guardrails (from research)

DoAvoid
Dark-first, ONE accent. Item photography carries the excitement.Promo-banner stacks & rotators above the fold.
Density adapts: low on home/opening, high on account tables.Live-ticker overflow competing with chat + leaderboards.
Rarity glow = the one decoration system (grey/blue/purple/gold).Badge inflation (SALE/HOT/NEW) on every card.
Trust surfaced passively — Provably Fair page + per-result links.Forced-open chat + floating support widget (V1 problem).
Progressive disclosure on Upgrade (stake → dial → catalog).Gradient + texture + neon layering all at once.

Type recommendation from research: Space Grotesk (display) + Inter (body/data) — confirm against the V1 Figma when shared. Colors above are working placeholders for the product. This document is themed with the COSS UI preset (Base UI / Cal.com): opaque alpha borders + bottom shadows, neutral palette, Geist headings.

07 Stage 2 — post-launch layer

Park these on the canvas as a separate dimmed region connected by dashed lines. Build path is MVP-first; these extend existing screens rather than replace them.

Post-launch features

Extensions and new surfaces deferred to Phase 2.
STAGE 2
  • Broadcast Mode — toggle on creation → chat announcement → voice or cam+voice (WebRTC) → spectator interaction layer
  • Bet Behind — spectators bet on a player/bot; 1v1 → x2, 1v1v1 → x3, etc.; real-time payout on outcome
  • Private Battles — password-protected rooms (for streamers)
  • Chat Rain — players trigger coin rain in site chat
  • Originals games — /crash · /mines · /plinko · /roulette · /dice
  • Mobile PWA — installable, push notifications, portrait layouts
  • Apple Pay + fiat deposits — 3rd-party provider
  • Full sound production — original audio (rain × packdraw blend)
Out of scope: No Pack Builder, no Steam trade offers, no gift cases (explicitly excluded by client). Turbo mode optional. 8-way battles are a stretch differentiator pending complexity assessment.
