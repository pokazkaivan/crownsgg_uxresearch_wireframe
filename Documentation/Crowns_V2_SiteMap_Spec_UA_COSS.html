Crowns — Карта сайту та специфікація User Flow

Повний план для канвасу FigJam та інтерактивного wireframe, який створюватимемо далі. Кожен екран нижче описано з його призначенням, компонентами й функціями, рівнем доступу, інтерактивними станами, модалками та всіма вхідними/вихідними зв'язками — тож кожна картка відповідає 1:1 вузлу, який ви розміщуєте на канвасі.

Реальні (IRL) фізичні предмети ~24 екрани MVP Шар Stage 2 позначено Авторизація: Email + Discord OAuth Каса: крипта + подарункові картки Виведення = тікет маркетплейсу

00 Як читати цей документ

Кожен фрейм екрана містить набір бейджів і чотири блоки контенту. На канвасі сприймайте кожен фрейм як один вузол/групу стікерів; кожен зв'язок — як стрілку між фреймами.

ROUTE

Реальний URL/сторінка, на яку користувач може потрапити або задіп-лінкуватись.

MODAL

Оверлей поверх поточної сторінки — без зміни URL. Повертає користувача туди, де він був.

PUBLIC

Доступно без акаунта. Працює на SEO та перше враження.

GATED

Потребує входу. Будь-яка дія тут викликає модалку авторизації, якщо користувач не залогінений.

ПЕРЕГЛЯД PUBLIC · ДІЯ GATED

Переглядати може будь-хто; цільова дія (відкрити / грати / надіслати) потребує входу.

STAGE 2

Після запуску. На канвасі розміщуйте як притлумлений/відкладений шар, поза шляхом збірки MVP.

Блоки фрейму:  Функції = компоненти й дії на екрані (• основні, ○ опційні/бажані).  Стани = кожен візуальний стан, який має покрити wireframe (порожній, завантаження, виграш, програш тощо).  Модалки = оверлеї, які цей екран може викликати.  Зв'язки = звідки користувач приходить (Вхід) і куди йде (Вихід).

01 Продукт і напрям дизайну

Базується на відповідях клієнта та конкурентному дослідженні. Crowns — це платформа кейсів із реальними фізичними предметами (не CS2-скіни, не предмети RuneScape). Естетична ціль — «тиха» частина спектра (спокій packy.gg / packdraw, ★★☆☆☆) — чітко й гладко, без відчуття скаму.

Сфера рішенняНапрям
Модель платформиРеальні фізичні предмети. Виграні предмети йдуть в інвентар/кошик → виконання вручну командою Crowns через тікет маркетплейсу (без введення адреси доставки на етапі виведення).
Механіки Day-1Відкриття кейсів (одиночне + мульти до x4), демо-спіни, кейс-баттли, апгрейдер (Borrow mode до 95%), нагороди (місячна гонка / тижневі челенджі / розіграш / безкоштовні кейси за рівнем).
БрендЗ нуля — відмова від гучного фіолетового V1. Бліда або темна, стримана схема. Один акцент; валюта/результати — єдині інші кольори.
АвторизаціяEmail + Discord OAuth. Age gate 18+.
КасаКрипта + подарункові картки з 1 дня. Apple Pay / фіат = Phase 2 (третя сторона).
ПрофільVIP-рівні + дашборд афіліатів важливі. Статистика — на запит. Публічна Вітрина виграшів, доступна за кліком на нікнейм у чаті/на сайті.
ЗвукМікс rain.gg × packdraw; оригінальні ассети — Stage 2.

02 Глобальна оболонка та навігація

Постійний каркас, що обгортає кожен маршрут. На канвасі намалюйте його один раз як «майстер-фрейм» і посилайтесь на нього; у wireframe це layout-компонент.

Оболонка застосунку

Верхня панель, баланс, згортуваний чат, футер — присутні на кожному маршруті.
ПОСТІЙНА
  • Логотип → Головна
  • Основна навігація: Кейси · Баттли · Апгрейд · Нагороди · Маркетплейс · Афіліати
  • Віджет балансу (золота валюта) + CTA Депозит → модалка депозиту
  • Залогінений: іконка кошика (з лічильником) → Кошик — тримає виграні предмети для продажу / доставки / відкладення
  • Без входу: кнопки Увійти / Реєстрація → модалки авторизації
  • Залогінений: меню аватара → Профіль, Інвентар, Гаманець, Афіліати, Налаштування, Вийти
  • Міні-індикатор рівня + XP → Профіль
  • Згортуваний сайдбар живого чату — прихований за замовчуванням на екранах відкриття та (Stage 2) Originals
  • Клік на будь-який нікнейм у чаті → Публічна вітрина
  • Точка входу Chat Rain STAGE 2
  • Футер: Provably Fair, Terms, Privacy, AML, FAQ, Підтримка
  • Підтримка: пасивне посилання/оверлей — без примусового плаваючого віджета
  • Age gate 18+ інтерстишал при першому візиті
Авторизація (Увійти/Реєстрація) · Депозит · Пошук · Оверлей підтримки

07

Апгрейдер

/upgrade
Механіка «ставка-і-множник» (розкладка: ставка ліворуч, радіальне ігрове поле праворуч). Прогресивне розкриття — ставка → множник → предмети-винагороди. Виграш потрапляє в Кошик, не одразу в інвентар.
ROUTEПЕРЕГЛЯД PUBLIC · ГРА GATED
  • Ваша ставка (ліворуч) — сума балансу + пресети (10/25/50/100% · Макс), або ставка предметами з інвентаря
  • Множник — x2 / x5 / x20 (задає шанс ≈ 1/множник)
  • Радіальна шкала шансу (праворуч) — дуга + тікер + живий % виграшу

07 Stage 2 — шар після запуску

Розмістіть це на канвасі як окрему притлумлену зону, з'єднану пунктиром. Шлях збірки — MVP-перш; ці фічі розширюють наявні екрани, а не замінюють їх.

Фічі після запуску

Розширення та нові поверхні, відкладені на Phase 2.
STAGE 2
  • Режим трансляції — перемикач при створенні → анонс у чаті → голос або камера+голос (WebRTC) → шар взаємодії зі спектаторами
  • Ставка за гравцем — спектатори ставлять на гравця/бота; 1v1 → x2, 1v1v1 → x3 тощо; виплата в реальному часі за результатом
  • Приватні баттли — кімнати під паролем (для стрімерів)
  • Chat Rain — гравці запускають «дощ монет» у чаті сайту
  • Ігри Originals — /crash · /mines · /plinko · /roulette · /dice
  • Мобільна PWA — встановлювана, push-сповіщення, портретні розкладки
  • Apple Pay + фіат-депозити — провайдер третьої сторони
  • Повне звукове продакшн — оригінальне аудіо (мікс rain × packdraw)
Поза скоупом: Без Pack Builder, без Steam trade offers, без gift cases (явно виключено клієнтом). Turbo mode опційний. 8-way баттли — стретч-диференціатор, залежить від оцінки складності.

Легенда — як читати
ROUTE

Реальний URL/сторінка, на яку користувач може потрапити або задіп-лінкуватись.

MODAL

Оверлей поверх поточної сторінки — без зміни URL. Повертає користувача туди, де він був.

PUBLIC

Доступно без акаунта. Працює на SEO та перше враження.

GATED

Потребує входу. Будь-яка дія тут викликає модалку авторизації, якщо користувач не залогінений.

ПЕРЕГЛЯД PUBLIC · ДІЯ GATED

Переглядати може будь-хто; цільова дія (відкрити / грати / надіслати) потребує входу.

STAGE 2

Після запуску. На канвасі розміщуйте як притлумлений/відкладений шар, поза шляхом збірки MVP.

Блоки фрейму: Функції = компоненти й дії (• основні, ○ опційні). Стани = кожен візуальний стан, який має покрити wireframe. Модалки = оверлеї, які цей екран може викликати. Зв'язки = звідки користувач приходить (Вхід) і куди йде (Вихід).
